Sass Básico

Personalización de Bootstrap con Sass

¿Qué es Sass?

Sass (Syntactically Awesome Style Sheets) es un preprocesador de CSS que te permite usar características que no existen en CSS puro:

  • Variables - Para reutilizar valores
  • Nesting - Anidar selectores
  • Mixins - Funciones reutilizables
  • Imports - Dividir CSS en archivos
  • Herencia - Compartir propiedades entre selectores

1. Variables

Ejemplo de variables

En lugar de repetir colores, defines una variable y la reutilizas:

// Sass $color-primary: #667eea; $color-secondary: #764ba2; $spacing-base: 1rem; .button { background-color: $color-primary; padding: $spacing-base; }

Esta sección usa las variables personalizadas definidas en custom.scss

2. Nesting (Anidamiento)

Estructura jerárquica

Puedes anidar selectores para que el código sea más legible:

// Sass .card { background: white; padding: 20px; &__title { font-size: 24px; color: blue; } &__body { margin-top: 10px; } &:hover { box-shadow: 0 4px 8px rgba(0,0,0,0.2); } } // Se compila a CSS: .card { background: white; padding: 20px; } .card__title { font-size: 24px; color: blue; } .card__body { margin-top: 10px; } .card:hover { box-shadow: 0 4px 8px rgba(0,0,0,0.2); }

El símbolo & representa el selector padre.

3. Mixins

Funciones reutilizables

Los mixins son como funciones que generan código CSS:

// Sass - Definir mixin @mixin button-style($bg-color, $text-color) { background-color: $bg-color; color: $text-color; padding: 10px 20px; border-radius: 5px; border: none; cursor: pointer; &:hover { opacity: 0.9; } } // Usar el mixin .btn-primary { @include button-style(#007bff, white); } .btn-danger { @include button-style(#dc3545, white); }

4. Personalizar Bootstrap con Sass

Sobrescribir variables de Bootstrap

Puedes cambiar los colores y estilos por defecto de Bootstrap:

// custom.scss // 1. Definir tus propias variables ANTES de importar Bootstrap $primary: #667eea; $secondary: #764ba2; $success: #10b981; $danger: #ef4444; $warning: #f59e0b; $info: #3b82f6; // 2. Importar Bootstrap @import "~bootstrap/scss/bootstrap"; // 3. Agregar tus estilos personalizados .mi-clase-custom { color: $primary; background: $secondary; }

Importante: Las variables deben definirse ANTES de importar Bootstrap para sobrescribirlas.

5. Compilar Sass

Proceso de compilación

Los navegadores no entienden Sass, hay que compilarlo a CSS:

# Instalar Sass npm install sass --save-dev # Compilar Sass a CSS npx sass scss/custom.scss styles.css # Watch mode (recompilar automáticamente al guardar) npx sass --watch scss/custom.scss:styles.css # En package.json agregar script: { "scripts": { "sass": "sass --watch scss/custom.scss:styles.css" } } # Ejecutar con: npm run sass

Tips Importantes

  • Variables: Usa $nombre-variable para definirlas
  • Nesting: No anides más de 3 niveles (puede ser difícil de mantener)
  • Mixins: Perfectos para código que se repite mucho
  • Import: Organiza tu código en múltiples archivos .scss
  • BEM: Sass funciona perfecto con la metodología BEM (.block__element--modifier)
  • Compilación: Siempre compila a CSS antes de usar en producción
  • Bootstrap: Define variables ANTES de importar Bootstrap para sobrescribir
